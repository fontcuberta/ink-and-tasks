## UX Audit — Ink & Tasks v2
Date: 2026-03-04
Environment: local (http://localhost:5174/)

### Summary

The Vue 3 migration is structurally solid with good design system discipline, proper auth flow, and ADHD-friendly gamification. However, there are **7 critical accessibility gaps** (missing focus traps in 3 modals, no keyboard access for Kanban cards and signature canvas, mobile sign-out missing accessible name), **4 critical interaction issues** (unsurfaced load errors, calendar event move errors silently fail), and a set of visual/copy refinements.

### Findings

Critical — 11 issues (broken accessibility, silent errors)
Suggestion — 17 issues (friction, inconsistency, polish)
Nice to have — 10 issues (copy, minor visual tokens)

---

#### 1. ProjectModal has no focus trap
Severity: Critical
Domain: Accessibility
View: Board
Observed: Tab key moves focus outside the modal into page content behind it.
Expected: Focus should be trapped inside the modal while open.
Suggested fix: Add `@keydown="trapFocus"` to the `.project-overlay`, identical to AuthModal's implementation.

---

#### 2. QuickAdd modal has no focus trap
Severity: Critical
Domain: Accessibility
View: Board
Observed: Same as ProjectModal — focus escapes.
Expected: Focus trapped while modal is open.
Suggested fix: Add focus trap logic matching AuthModal.

---

#### 3. EventModal has no focus trap
Severity: Critical
Domain: Accessibility
View: Calendar
Observed: Same — no focus management.
Expected: Focus trapped.
Suggested fix: Same pattern.

---

#### 4. Consent print overlay lacks dialog semantics + focus trap
Severity: Critical
Domain: Accessibility
View: Consent
Observed: Print overlay has no `role="dialog"`, no `aria-modal`, no focus trap.
Expected: Full dialog semantics and trapped focus.
Suggested fix: Add `role="dialog" aria-modal="true" aria-label="Consent record"` to `.print-overlay`, add `@keydown` focus trap.

---

#### 5. Kanban cards not keyboard-accessible
Severity: Critical
Domain: Accessibility
View: Board
Observed: Cards are `<div @click>`; not reachable or activatable via keyboard.
Expected: Keyboard users should be able to Tab to cards and press Enter/Space to open.
Suggested fix: Add `tabindex="0"` and `@keydown.enter="openProject(element)" @keydown.space.prevent="openProject(element)"` to card divs. Or use `<button>` wrappers.

---

#### 6. Kanban drag-and-drop has no keyboard alternative
Severity: Critical
Domain: Accessibility
View: Board
Observed: Cards can only be moved between columns via mouse drag.
Expected: Keyboard users should be able to move cards with arrow keys or a column-change dropdown.
Suggested fix: Add a small dropdown or keyboard shortcut (e.g., Arrow Left/Right when card is focused) to change a card's column.

---

#### 7. SignaturePad has no keyboard/text alternative
Severity: Critical
Domain: Accessibility
View: Consent
Observed: Signature canvas is mouse/touch only; no `aria-label`, no keyboard alternative.
Expected: Canvas should have `role="img"` with `aria-label`, plus a text-based fallback.
Suggested fix: Add `role="img" aria-label="Signature input. Draw with mouse or touch."` to the canvas, and add a "Type your name" text input as an alternative.

---

#### 8. Mobile sign-out button missing accessible name
Severity: Critical
Domain: Accessibility
View: Global
Observed: The mobile topbar sign-out button (`btn--signout-sm`) shows only an icon, no text and no `aria-label`.
Expected: Should have `aria-label="Sign out"`.
Suggested fix: Add `aria-label="Sign out"` to the button in App.vue template.

#### 9. useProjects load errors not surfaced to user

### What's working well

- **Auth modal** has proper focus trap, `role="dialog"`, `aria-labelledby`, `aria-modal`, `role="alert"` on errors — gold standard.
- **Toast container** has `aria-live="polite"` and `role="status"` — screen readers announce notifications.
- **ProgressRing** has `role="img"` with descriptive `aria-label`.
- **`.sr-only` utility** is available and used for hidden labels.
- **`prefers-reduced-motion`** respected for pulse animation, progress ring transition, gamification sounds, and confetti.
- **Design system discipline** — fonts, CSS custom properties, border-radius tokens, shadow tokens used consistently.
- **Quick-add** is genuinely 2-tap: name + submit. Photo is optional. Meets the ADHD-friendly target.
- **Gamification** — confetti, whoosh sound, success chime, streak counter, pulsing next-action glow. All respect reduced-motion.
- **Responsive layout** — sidebar collapses to hamburger on mobile, topbar appears, views adapt.
- **Navigation** uses `router-link` with `.active` class for clear current-view indication.
- **Toast copy** is concise and reassuring ("Project created", "Welcome back!", "Signed out").
- **Form placeholders** are descriptive and helpful ("e.g. forearm", "List any known allergies...").
- **Style badge system** covers 8+ tattoo styles with distinct colour mapping.
