# Accessibility Audit — Ink & Tasks
**Date:** 2026-03-05 | **Standard:** WCAG 2.2 Level AA

## Summary
Strong foundation: all modals have `role="dialog"` + focus traps, toasts use `aria-live`, kanban cards have keyboard arrow-key movement, signature pad has typed-name fallback, `prefers-reduced-motion` is respected. 7 critical issues remain around focus indicators, hidden interactive elements, and missing ARIA semantics.

## Findings
**Critical — 7 | Suggestion — 11 | Nice to have — 3**

### Critical
1. **No custom focus indicator** on buttons, cards, nav links — invisible on dark backgrounds
2. **Input focus ring contrast too low** (~1.4:1 with gold glow at 18% opacity)
3. **Image action buttons invisible to keyboard** — `opacity:0` on hover only, no `:focus-within`
4. **File inputs inaccessible** — `display:none` removes from tab order
5. **Kanban columns lack semantic ARIA roles** — no `role="group"` or `aria-label`
6. **No live region announcing kanban card moves** — sighted users get confetti, screen readers get nothing
7. **Consent checkboxes not in fieldset/legend** — no programmatic grouping

### Suggestions
8. Placeholder text fails contrast (~2.7:1)
9. Dotwork badge text fails contrast (~3.7:1)
10. Toast auto-dismiss doesn't pause on hover
11. Modal/toast transitions not disabled for reduced-motion
12. No document.title update on route changes
13. Missing autocomplete attributes on client info fields
14. Auth error not linked to input via aria-describedby
15. Loading containers missing aria-busy
16. Sound effects lack independent toggle
17. Successful card moves not announced to screen readers
18. No skip-to-content link

### Nice to have
19. Decorative icons inconsistently marked aria-hidden
20. Muted text contrast is borderline (4.51:1)
21. Modals use aria-label instead of aria-labelledby

## Passing checks
- All form inputs have `<label>` with matching for/id
- AuthModal uses correct autocomplete attributes
- Badges and legend convey info via text, not colour alone
- All major colour pairs pass 4.5:1 contrast
- Kanban cards have Arrow Left/Right + Enter/Space keyboard nav
- All five modals have role="dialog", aria-modal="true", and focus traps
- Signature pad has role="img", aria-label, and typed-name fallback
- prefers-reduced-motion disables confetti, pulse-glow, progress ring
- html lang="en" present
